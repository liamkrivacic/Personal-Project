UNSW SumoBots is exactly what it sounds like: autonomous robots shoving each other around a
white-bordered ring, last one inside the line wins. Simple to describe, surprisingly hard to do
well: your robot has to find the opponent, commit to a push, and *not* drive itself out of the
ring in the process, all with no human input once the match starts. The advanced stream runs more
capable hardware and gives you enough room to actually have a strategy.

I won the advanced stream in August 2024, and most of why I won had nothing to do with the
single match and everything to do with the sixteen chassis revisions behind it.

## Mechanical design

I designed the chassis in Fusion 360, and it took 16 revisions to get right. The early ones were
about nailing the basic layout: a low-profile wedge with just enough floor area to mount three
ultrasonic sensors at the front *without* letting them poke past the robot's outline, because
anything sticking out is something an opponent can catch and snap off mid-match. A lot of those
revisions were me learning that lesson the hard way and tucking things back in.

Key manufacturing decisions:
- **Laser-cut acrylic** produced the five flat structural panels (floor, two sides, back, roof)
  with precise tab-and-slot joints for self-fixturing during assembly.
- **3D printing** handled the sensor bracket at the front: the geometry needed to hold three
  ultrasonic sensors at specific outward angles, which was too complex for flat sheet work.
- **Low centre of mass**: the battery and Arduino were positioned as low as possible in the
  chassis to reduce the risk of the robot tipping backward when pushing hard.
- **Wedge profile**: the angled front face makes it harder for the opponent to get underneath,
  which was the most common failure mode observed in earlier competition rounds.

## Electronics and sensors

I gave it three ultrasonic distance sensors covering the forward, front-right and front-left
arcs. That spread is enough for the robot to spot an opponent and lean its turn toward the
strongest return, so instead of charging blindly forward, it homes in on a target it can only
half-see. That homing behaviour is what won most of the matches.

Under the chassis, IR line sensors watch for the white boundary. The moment one trips, the robot
drops whatever it was doing, runs an escape manoeuvre, and only then goes back to hunting:
self-preservation overrides aggression, always.

I wired the sensors on separate trigger/echo pin pairs (A, B, C) specifically so I could read all
three in a single loop pass instead of blocking on one ultrasonic ping at a time. In a pushing
match, a sluggish sensor loop is a lost match.

## Control logic

The firmware juggles two jobs that are constantly in tension: staying inside the ring
(safety-critical) and catching the opponent (performance-critical). I hard-coded the priority
order into the main loop so the robot never has to "decide" which matters more under pressure:

1. Read the IR sensor. If the boundary line is detected, reverse then turn away from the
   edge before doing anything else.
2. Read all three ultrasonic sensors in sequence.
3. If the front sensor reads an opponent within range, drive forward.
4. If a side sensor reads an opponent, turn toward the stronger return.
5. If no opponent is detected, execute a slow rotation search to sweep the arena.

That `DISTANCE_TRIGGER` constant, 70 cm in the final firmware, was pure trial and error during
practice runs. Set it too low and the robot ignores an opponent sitting in its peripheral sensor;
too high and it starts twitching at the arena walls from the neutral zone. 70 cm was the number
where it reacted to opponents and not to architecture.

I drive each side's motors independently over PWM, which lets the robot turn gradually by mixing
forward and stop instead of having to slam into reverse to change heading. The 5-second startup
delay runs as a one-shot flag on the first loop pass, so it satisfies the competition rule without
freezing the serial monitor I was using to debug.

## What the build demonstrated

More than anything, SumoBots taught me that this is a systems-integration problem wearing a
robot costume. The mechanical, electronic and firmware decisions all reach into each other:
chassis geometry dictates where the sensors can sit, sensor positions dictate the firmware logic,
and how aggressive that firmware is decides whether the chassis survives a hard hit.

I didn't win by perfecting any one of those layers in isolation. I won by iterating on all three
together, revision after revision, until they stopped fighting each other, which is the same way
the harder engineering I've done since actually gets finished.

## Source code

<CodeBlock title="V4.2.ino - final competition firmware">

```cpp
// Motor 1
#define in1 10
#define in2 11

// Motor 2
#define in3 5
#define in4 6

// Distance sensor A (front)
#define trigPinA 9
#define echoPinA 8

// Distance sensor B (front-right)
#define trigPinB 3
#define echoPinB 2

// Distance sensor C (front-left)
#define trigPinC 7
#define echoPinC 4

// IR line sensor
#define IR 12

#define TURN_SPEED 255
#define FORWARD_SPEED 255
#define DISTANCE_TRIGGER 70

long durationA, durationB, durationC;
int distanceA, distanceB, distanceC;
int in1Speed = TURN_SPEED;
int in2Speed = 0;
int in3Speed = 0;
int in4Speed = TURN_SPEED;
int alert;
int mode;
char command;
int wait = 0;

void setup() {
  pinMode(in1, OUTPUT); pinMode(in2, OUTPUT);
  pinMode(in3, OUTPUT); pinMode(in4, OUTPUT);
  pinMode(trigPinA, OUTPUT); pinMode(echoPinA, INPUT);
  pinMode(trigPinB, OUTPUT); pinMode(echoPinB, INPUT);
  pinMode(trigPinC, OUTPUT); pinMode(echoPinC, INPUT);
  pinMode(IR, INPUT);
  Serial.begin(9600);
}

void loop() {
  // 5-second start delay (one-shot)
  if (wait == 0) {
    delay(5000);
    wait = 1;
  }

  // Read ultrasonic sensors
  digitalWrite(trigPinA, LOW); delayMicroseconds(2);
  digitalWrite(trigPinA, HIGH); delayMicroseconds(10); digitalWrite(trigPinA, LOW);
  durationA = pulseIn(echoPinA, HIGH);
  distanceA = durationA * 0.34 / 20;

  digitalWrite(trigPinB, LOW); delayMicroseconds(2);
  digitalWrite(trigPinB, HIGH); delayMicroseconds(10); digitalWrite(trigPinB, LOW);
  durationB = pulseIn(echoPinB, HIGH);
  distanceB = durationB * 0.34 / 20;

  digitalWrite(trigPinC, LOW); delayMicroseconds(2);
  digitalWrite(trigPinC, HIGH); delayMicroseconds(10); digitalWrite(trigPinC, LOW);
  durationC = pulseIn(echoPinC, HIGH);
  distanceC = durationC * 0.34 / 20;

  alert = digitalRead(IR);

  // IR line detection: reverse then turn (highest priority)
  if (alert == HIGH) {
    analogWrite(in1, 0); analogWrite(in2, FORWARD_SPEED);
    analogWrite(in3, 0); analogWrite(in4, FORWARD_SPEED);
    delay(1200);
    in3Speed = 0; in4Speed = TURN_SPEED;
    in1Speed = TURN_SPEED; in2Speed = 0;
    mode = 5;
  }
  // No opponent in range: slow rotation search
  else if (distanceA > DISTANCE_TRIGGER && distanceB > DISTANCE_TRIGGER && distanceC > DISTANCE_TRIGGER) {
    delayMicroseconds(500);
    mode = 1;
  }
  // Front sensor: drive forward
  else if (distanceA <= DISTANCE_TRIGGER) {
    delay(distanceA * 0.45);
    in1Speed = FORWARD_SPEED; in2Speed = 0;
    in3Speed = FORWARD_SPEED; in4Speed = 0;
    mode = 2;
  }
  // Right sensor: turn right
  else if (distanceB <= DISTANCE_TRIGGER) {
    in1Speed = 0; in2Speed = TURN_SPEED;
    in3Speed = TURN_SPEED; in4Speed = 0;
    mode = 3;
  }
  // Left sensor: turn left
  else if (distanceC <= DISTANCE_TRIGGER) {
    in1Speed = TURN_SPEED; in2Speed = 0;
    in3Speed = 0; in4Speed = TURN_SPEED;
    mode = 4;
  }

  analogWrite(in1, in1Speed); analogWrite(in2, in2Speed);
  analogWrite(in3, in3Speed); analogWrite(in4, in4Speed);

  // Serial debug interface
  if (Serial.available() > 0) {
    command = Serial.read();
    if (command == 'm') Serial.println(mode);
    else if (command == 'r') {
      Serial.println(in3Speed);
      Serial.println(in4Speed);
    }
    else if (command == 'd') {
      digitalWrite(trigPinA, LOW); delayMicroseconds(2);
      digitalWrite(trigPinA, HIGH); delayMicroseconds(10); digitalWrite(trigPinA, LOW);
      durationA = pulseIn(echoPinA, HIGH);
      distanceA = durationA * 0.34 / 20;
      Serial.print("DistanceA: "); Serial.println(distanceA);
    }
  }
}
```
